Validates that the key used in the unit file matches a known value for the section (e.g., [Unit], [Install], [Service]). Consistent with the documentation, this inspection ignores keys or sections that start with X-.

The valid keys depend on their section and are generated via scraping the documentation.

For more information on the general behaviour see: systemd.unit(5).

Individual section options are described both in their respective man pages (e.g., systemd.service(5)), but additional options are documented in shared sections (e.g., systemd.resource-control(5)).
